## 다음 단계 후보

UI 목업이 완성되었으니 Stage 2로 넘어가는 옵션을 정리합니다. 순서/우선순위는 사용자가 정합니다.

### A. 데이터 통합 + localStorage 영속화 ⭐ 추천

**왜**: 가장 큰 단점("새로고침하면 다 사라짐", "화면 간 데이터 분리")을 한 번에 해결.

**무엇**: 손님/적립 내역/가게 설정을 단일 store로 통합, localStorage에 저장. 사장님이 적립한 내역이 손님 조회 화면에 실시간 반영.

**ADR 필요**: 데이터 모델 + 영속화 전략 (localStorage vs IndexedDB).

### B. 가게 설정 화면

**왜**: ADR-0003/0004에서 후속 결정으로 미뤄둠. 현재 가게 이름·적립률·PIN이 코드 상수.

**무엇**: 대시보드의 "가게 정보" 카드를 편집 가능하게. PIN 변경 흐름.

### C. 리워드 사용 흐름 (포인트 차감)

**왜**: 적립만 있고 사용이 없어 절반의 흐름. 손님 조회의 "🎁 포인트 사용" 버튼이 동작하지 않음.

**무엇**: 사장님 손님 조회 → 5,000P 사용 → 손님 잔액 차감 → 손님 화면에 "사용 완료" 반영.

### D. 실제 라우팅 도입 ([[adr-routing]])

**왜**: 화면이 6개로 늘었고, 데모 토글의 한계가 명확해짐.

**무엇**: React Router 도입, 손님 URL과 사장님 URL 분리. ADR-0006 Superseded 처리.

---

**제 추천 순서**: A → C → B → D. A를 먼저 해야 C가 의미 있고, B는 그 다음 자연스러운 흐름.
