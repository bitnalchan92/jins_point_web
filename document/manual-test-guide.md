# 수동 테스트 가이드

UI 목업 단계의 화면들을 어떻게 직접 확인하고 검증할지 정리한 문서입니다. 자동화 테스트 대신 **눈으로 보면서 빠르게 한 바퀴 도는 체크리스트**입니다.

화면이 추가되거나 동작이 바뀌면 이 문서를 같이 갱신해주세요.

## 1. 준비

```bash
npm install
npm run dev
```

브라우저: **http://localhost:5173/**

권장 환경:

- 브라우저: Chrome 또는 Safari 최신 버전
- **손님 화면** (`/`): 모바일 뷰포트. Chrome DevTools에서 `Cmd+Shift+M` → iPhone 15 Pro (393×852) 권장. 화면 안에 phone frame(430px, min-h 740px)이 렌더링됩니다.
- **사장님 화면** (`/admin`): 데스크탑 뷰포트(1,024px+). 컨테이너 최대 1,180px으로 POS 와이드 레이아웃 표시.

## 2. URL 라우팅

데모 토글 없이 **URL로 화면을 분리**합니다.

| URL | 화면 |
|-----|------|
| `/` | 손님 화면 — 전화번호 입력 → 포인트 조회 |
| `/admin` | 사장님 화면 — PIN 로그인 → 포인트·손님 관리·대시보드 |
| 그 외 | `/`로 리다이렉트 |

## 3. 시나리오: 사장님 로그인

진입: `/admin`

### 3.1 초기 진입

- [ ] 배경이 크림 그라디언트 (밝고 따뜻한 톤)
- [ ] 가운데 정렬 카드 (max-w 400px)
- [ ] 상단에 🍙 브랜드 로고 + "달콤한 진스쿡" 가게 이름
- [ ] "사장님 로그인" 제목 + 안내 문구
- [ ] PIN 점 4개 (비어있음, 옅은 베이지 원)
- [ ] 아래 3×4 온스크린 키패드 (1~9, C, 0, ⌫)
- [ ] 상단에 **DEMO · PIN 1234 자동 입력** 버튼
- [ ] "로그인" 버튼 비활성 (회색)

### 3.2 PIN 입력 동작

- [ ] 키패드 숫자 클릭 → 점 하나씩 채워짐
- [ ] 4자리 입력 완료 → "로그인" 버튼 활성
- [ ] C 버튼 → 전체 지워짐
- [ ] ⌫ 버튼 → 마지막 자리 하나 삭제
- [ ] 4자리 초과 입력 안 됨

### 3.3 잘못된 PIN

- [ ] PIN `0000` 입력 후 "로그인" 클릭 → 점 영역 **흔들리는 애니메이션** (jp-shake)
- [ ] 오류 메시지 "PIN이 일치하지 않아요" 표시
- [ ] PIN 자동 비워짐, 다음 입력 시 에러 사라짐

### 3.4 로그인 후 흐름

- [ ] **상단 탭바** 등장: 🍙 로고 + [☕ 포인트 / 👥 손님 관리 / 📊 대시보드]
- [ ] 포인트 탭이 기본 활성
- [ ] 사장님 모드 전체가 1,180px 와이드 POS 레이아웃
- [ ] `/admin` URL 유지 상태에서 브라우저 새로고침 → 로그인 화면으로 복귀 (세션 메모리만 사용)

## 4. 시나리오: 사장님 포인트 탭 (☕ 포인트)

진입: `/admin` 로그인 후 포인트 탭

### 4.0 모드 토글

- [ ] 화면 상단에 "🍙 적립 / 🎁 사용" 필 토글
- [ ] 활성 탭: 브랜드 옐로우 배경 + 잉크 텍스트
- [ ] 탭 전환 시 모든 입력이 초기화되고 STEP 1 (손님 조회)로 돌아감

### 4.1 STEP 1 — 손님 조회 (4자리 뒷번호)

왼쪽 패널:

- [ ] "포인트 적립" 또는 "포인트 사용" 제목
- [ ] 4자리 핀패드 형식 — 진행 점 4개 (비어있음/채워짐)
- [ ] 숫자 키패드로 4자리 입력
- [ ] 4자리 미완성 시 "손님 조회" 버튼 비활성
- [ ] 4자리 완료 후 "손님 조회" 클릭 → 매칭 결과에 따라 분기

오른쪽 패널 (미리보기):

- [ ] 손님을 아직 선택하지 않은 상태에서 🧾 또는 🎁 안내 카드 표시

### 4.2 STEP 2 — 결과 분기

**1명 매칭**

- [ ] STEP 3 (금액 입력)으로 자동 이동 (손님 선택 생략)

**2명 이상 매칭**

- [ ] "손님 선택" 화면 표시
- [ ] 뒷번호 `XXXX` — N명 매칭 안내
- [ ] 각 손님 행: 이름 + 전체 전화번호 + 보유 포인트 + 방문 횟수
- [ ] 행 클릭 → STEP 3으로 이동
- [ ] "← 다시" 버튼 → STEP 1로 돌아감

**0명 매칭**

- [ ] 🔍 + "뒷번호 XXXX인 손님을 찾을 수 없어요" 빈 상태 표시
- [ ] "다시 입력" 버튼 → STEP 1으로 복귀

### 4.3 STEP 3 — 금액 입력 (적립 모드)

| 테스트 케이스 | 뒷번호 | 예상 매칭 |
|-------------|--------|---------|
| 단골 자동 선택 | `7788` | 김서연 1명 → 자동 진입 |
| 복수 선택 | `7788`로 2명 추가된 경우 | 선택 UI |

- [ ] "← 다시" 버튼으로 STEP 1 복귀 가능
- [ ] 선택된 손님 이름 표시 + 보유 포인트 (작은 텍스트)
- [ ] 결제 금액 표시 영역 (`0 원`)
- [ ] 빠른 칩: `+1,000` / `+5,000` / `전체삭제` (누적 방식)
- [ ] 숫자 키패드 3×4 (1~9, 00, 0, ⌫)
- [ ] 오른쪽 미리보기 카드: 손님 이름·보유포인트·방문횟수·적립 예정 포인트 (+)

**적립 실행 (김서연, 5,400원)**

- [ ] 금액 입력 → 미리보기에 `+270P` (5% 적용)
- [ ] 적립 예정 안내 배너: "적립 예정: +270P (5.0% 적용)"
- [ ] "포인트 적립하기" 버튼 활성 (잉크 다크)
- [ ] 버튼 클릭 시:
  - [ ] 하단 토스트: 🍙 "김서연 · +270P 적립 완료" · "포인트가 적립되었어요"
  - [ ] 2.6초 후 토스트 사라짐
  - [ ] STEP 1으로 초기화
  - [ ] 최근 내역에 새 항목 추가 (+270P)

### 4.4 STEP 3 — 금액 입력 (사용 모드)

- [ ] 선택된 손님 이름 아래 **보유 포인트 배너** (옐로우 그라디언트, 큰 숫자 표시)
- [ ] 금액 단위: "P"
- [ ] 빠른 칩: `+1,000P` / `+3,000P` / `전체삭제` (누적 방식)
- [ ] 힌트 배너: "1,000P 단위로만 사용 가능해요"

**1,000P 단위 검증**

| 입력 | 기대 |
|------|------|
| `3000` | 정상 — "포인트 사용하기" 활성 |
| `1500` | ⚠️ "1,000P 단위로 입력해주세요" 경고 + 버튼 비활성 |

**포인트 부족 검증**

- [ ] 보유 포인트 초과 입력 → "⚠️ 포인트가 부족해요 (최대 XXXP)" 경고 + 버튼 비활성

**사용 실행 (김서연, 3,000P)**

- [ ] "포인트 사용하기" 버튼: 붉은색 활성
- [ ] 버튼 클릭 시:
  - [ ] 하단 토스트: 🎁 "김서연 · -3,000P 사용 완료" · "잔여 포인트 420P"
  - [ ] STEP 1으로 초기화
  - [ ] 최근 내역에 -3,000P (붉은색) 추가

## 5. 시나리오: 사장님 손님 관리 탭 (👥 손님 관리)

진입: `/admin` 로그인 후 손님 관리 탭

- [ ] 검색 입력창 (placeholder: "이름 또는 전화번호로 검색")

- [ ] 손님 카드 그리드 5명 (포인트 높은 순):
  - `이준호` · 5,180P · 24회
  - `김서연` · 3,420P · 18회
  - `정하윤` · 2,890P · 13회
  - `박지우` · 1,240P · 7회

- [ ] `7788` (전화번호 부분) → 결과 1건 (김서연)
- [ ] `이준` (이름 부분) → 결과 1건 (이준호)
- [ ] `9999` (일치 없음) → 빈 상태 메시지
- [ ] 검색어 지우면 전체 5명 복귀

### 5.3 카드 펼침

- [ ] 카드 클릭 → 상세 펼침
  - [ ] 무료 메뉴까지 남은 포인트

  - [ ] 최근 내역 (날짜 + 포인트 증감만 표시, 메뉴명 없음)
- [ ] 다른 카드 클릭 → 이전 카드 자동 접힘

### 5.4 신규 손님 추가

- [ ] "+ 신규 손님" 버튼 클릭 → 인라인 폼 표시 (이름 + 전화번호)
- [ ] 버튼이 "취소"로 변경
- [ ] 이름 미입력 → "이름을 입력해주세요" 에러
- [ ] 전화번호 10자리 미만 → "올바른 전화번호를 입력해주세요" 에러
- [ ] 이미 등록된 번호 → "이미 등록된 전화번호예요" 에러
- [ ] 정상 입력 후 "등록하기" 클릭:
  - [ ] 폼 즉시 사라짐
  - [ ] 하단 토스트: 🌱 "[이름] 님 등록 완료" · "신규 손님으로 추가되었어요"
  - [ ] 손님 카드 목록에 신규 손님 즉시 등장 (0P, 0회 방문)
- [ ] 전화번호 자동 포맷: `01012345678` → `010-1234-5678`

### 5.5 적립 후 실시간 반영

## 6. 시나리오: 사장님 대시보드 탭 (📊 대시보드)

진입: `/admin` 로그인 후 대시보드 탭

### 6.1 KPI 카드

- [ ] 3개 카드 (가로 균등):
  - ☕ **오늘 적립**: 이번 세션에서 처리한 적립 건수
  - 👥 **손님 수**: 전체 등록 손님 수 (신규 추가 시 실시간 반영)
  - ⭐ **총 사용가능 포인트**: 모든 손님의 포인트 합산 (실시간 반영)

### 6.2 두 번째 라인

**가게 정보 (좌)**

- [ ] 가게명: 달콤한 진스쿡
- [ ] 업종: 김밥·샌드위치 전문점
- [ ] 적립률 + 수정 버튼
- [ ] 무료 메뉴 기준: 5,000P
- [ ] 누적 단골 수

**단골 TOP 3 (우)**

- [ ] 포인트 높은 순 TOP 3 (실시간 반영)
  - 1위 배지: 브랜드 옐로우
  - 2·3위 배지: 옅은 베이지

### 6.3 적립률 수정

- [ ] "수정" 클릭 → 숫자 입력 필드 + % + 저장/취소
- [ ] `10` 입력 후 "저장" → "결제금액의 10.0%"로 갱신
- [ ] 포인트 탭으로 이동해서 5,400원 입력 시 `+540P` 표시
- [ ] Enter 키 저장, Escape 키 취소
- [ ] 빈값·0 이하: "저장" 비활성

### 6.4 로그아웃

- [ ] 하단에 가로 풀너비 붉은색 "로그아웃" 버튼
- [ ] 클릭 → 로그인 화면 복귀, 상단 탭바 사라짐
- [ ] 다시 로그인하면 포인트 탭으로 시작
