# 수동 테스트 가이드

UI 목업 단계의 화면들을 어떻게 직접 확인하고 검증할지 정리한 문서입니다. 자동화 테스트 대신 **눈으로 보면서 빠르게 한 바퀴 도는 체크리스트**입니다.

화면이 추가되거나 동작이 바뀌면 이 문서를 같이 갱신해주세요.

## 1. 준비

```bash
npm install
npm run dev
```

브라우저: **http://localhost:5173/**

권장 환경:

- 브라우저: Chrome 또는 Safari 최신 버전
- 디바이스: **모바일 뷰포트** 우선. Chrome DevTools에서 `Cmd+Shift+M` → iPhone 15 Pro (393×852) 권장
- 데스크탑 너비에서는 컨테이너가 `max-w-md` (≈448px)로 가운데 정렬됩니다

## 2. 데모 모드 스위처

페이지 **최상단의 작은 띠**에 데모 토글이 있습니다. (실 제품의 UX가 아니라 두 화면을 한 dev 환경에서 비교하기 위한 장치입니다 — [ADR-0006](./adr/0006-demo-mode-switcher.md))

| 토글 | 보여지는 화면 |
|------|----------------|
| 🧑‍🍳 사장님 | `OwnerLoginScreen` → (PIN 인증 후) 하단 탭바 + 적립/조회/대시보드 |
| 📱 손님 | `CustomerLandingScreen` → (전화번호 입력 후) `CustomerPointScreen` |

**확인할 것**

- [ ] 두 버튼이 보이고, 활성 모드가 다크 커피색으로 표시된다
- [ ] 토글을 누르면 즉시 화면이 바뀐다
- [ ] 손님 모드로 다시 전환하면 항상 **랜딩 화면**부터 다시 시작한다 (이전 입력은 비워짐)
- [ ] 사장님 모드는 **로그인 상태가 유지된다** (한 번 로그인하면 손님 모드로 갔다 와도 그대로). 명시적으로 로그아웃 버튼을 눌러야 로그인 화면으로 돌아간다

## 3. 시나리오: 사장님 로그인

진입: 토글 → 🧑‍🍳 사장님

### 3.0.1 초기 진입

- [ ] 상단에 다크 커피 그라디언트 Hero 카드 (🧑‍🍳 + "진의 카페 · 사장님 모드")
- [ ] "사장님 로그인" 제목 + 안내 문구
- [ ] PIN 박스 4개가 모두 비어있는 상태 (옅은 베이지)
- [ ] PIN 입력 자동 포커스
- [ ] "로그인" 버튼 비활성 (회색)
- [ ] 하단 **DEMO 자동 입력 칩** (PIN 1234)

### 3.0.2 PIN 입력 동작

- [ ] 숫자 1자 입력 → 첫 번째 박스가 카라멜색으로 채워짐 + 살짝 커지는 애니메이션
- [ ] 4자리 입력 완료 → 모든 박스 카라멜, 로그인 버튼 활성 (다크 커피색)
- [ ] 숫자가 아닌 문자 입력 시 무시
- [ ] maxLength 4 (5자리 입력 안 됨)

### 3.0.3 잘못된 PIN

- [ ] PIN `0000` 입력 후 로그인 → "PIN이 일치하지 않아요" 카라멜 메시지
- [ ] PIN 자동으로 비워지고 첫 박스로 포커스 복귀
- [ ] 다음 글자 입력 시 에러 메시지 사라짐

### 3.0.4 데모 자동 입력

- [ ] 하단 데모 칩 클릭 → 입력란에 `1234` 자동 채움
- [ ] 4개 박스가 모두 카라멜로 채워지고 버튼 활성
- [ ] "로그인" 클릭 → 사장님 메인(적립 탭)으로 진입

### 3.0.5 로그인 후 흐름

- [ ] 화면 하단에 **탭바** 등장 (☕ 적립 / 👥 손님 조회 / 📊 대시보드)
- [ ] 적립 탭이 기본 활성 (카라멜색)
- [ ] 토글로 손님 모드 갔다가 다시 사장님 모드 와도 **로그인 상태 유지**, 마지막 탭으로 복귀

## 3. 시나리오: 사장님 적립 화면 (탭 ☕)

진입: 사장님 로그인 후 ☕ 적립 탭

### 3.1 초기 상태 (입력 전)

- [ ] 헤더에 ☕ 아이콘 + "진의 카페 · 사장님 모드"
- [ ] 우측 ⚙ 설정 버튼
- [ ] 제목 "포인트 적립" + 안내 문구
- [ ] 전화번호 입력 / 결제 금액 입력 카드 (둘 다 비어있음, "확인됨" 안 보임)
- [ ] 적립 미리보기 카드: `0 P` (희미한 색)
- [ ] 메인 버튼 비활성: 회색 "정보를 입력해주세요"
- [ ] 최근 적립 리스트에 목업 3건 (`010-****-7788`, `010-****-4412 NEW`, `010-****-1290`)

### 3.2 단골 손님 적립

| 입력 | 값 |
|------|-----|
| 전화번호 | `01023457788` |
| 결제 금액 | `5400` |

기대:

- [ ] 전화번호 자동 포맷팅 → `010-2345-7788`
- [ ] 결제 금액 자동 콤마 → `5,400`
- [ ] 두 카드 우측에 초록색 "확인됨"
- [ ] 미리보기 카드: 카라멜 그라디언트, `270 P` (큰 글씨)
- [ ] 신규 손님 뱃지 **없음**
- [ ] 메인 버튼 활성: 카라멜색 "☕ 포인트 적립하기"
- [ ] 버튼 클릭 시:
  - [ ] 화면 하단 중앙에 다크 토스트: ☕ + "적립 완료" + "+270P"
  - [ ] 약 2.4초 후 사라짐
  - [ ] 입력 필드 비워짐
  - [ ] 최근 적립 리스트 최상단에 `010-****-7788 · 5,400원 · 방금 · +270P` 추가

### 3.3 신규 손님 적립

| 입력 | 값 |
|------|-----|
| 전화번호 | `01055551111` (시스템에 등록 안 된 임의 번호) |
| 결제 금액 | `12500` |

기대:

- [ ] 미리보기 카드: `625 P`
- [ ] 우측에 🌱 **신규 손님** 초록 뱃지
- [ ] 적립 후 토스트: "신규 손님 첫 적립" + "+625P"
- [ ] 리스트 최상단에 `NEW` 뱃지 같이 표시

### 3.4 유효성 / 가드

- [ ] 전화번호 9자리 이하 (예: `010234567`): 버튼 비활성, "확인됨" 안 보임
- [ ] 결제 금액 1,000원 미만 (예: `500`): 카드 우측에 "1,000원 이상" 힌트, 버튼 비활성
- [ ] 전화번호 11자리 초과 입력: 11자리에서 잘림
- [ ] 결제 금액 7자리 초과: 잘림

### 3.5 자동 포맷팅

- [ ] 전화번호: `01012345678` → `010-1234-5678` (자동 하이픈)
- [ ] 결제 금액: `12500` → `12,500` (천단위 콤마)
- [ ] 숫자가 아닌 문자 입력: 무시됨

## 4. 시나리오: 사장님 손님 조회 (탭 👥)

진입: 사장님 로그인 후 👥 손님 조회 탭

### 4.1 초기 상태 (검색 전)

- [ ] 검색 입력 카드 (비어있음, "010-1234-5678" placeholder)
- [ ] 그 아래 "단골 손님" 섹션 + "잔액 높은 순" 라벨
- [ ] 단골 손님 리스트 6명 표시:
  - `010-****-3434` 12,400P (1위)
  - `010-****-4444` 8,120P
  - `010-****-1234` 5,200P
  - `010-****-7788` 3,420P
  - `010-****-1290` 1,850P
  - `010-****-4412` 700P + NEW 뱃지

### 4.2 검색

- [ ] `7788` 입력 → 결과 1건 (`010-****-7788`만 남음)
- [ ] "검색 결과 1건" 라벨
- [ ] 검색창 우측 "지우기" 버튼이 나타남 → 클릭 시 전체 리스트로 복귀
- [ ] `9999` 같은 일치 없는 번호 → empty state (🔍 + "일치하는 손님이 없어요")
- [ ] empty state에 검색어가 그대로 표시됨

### 4.3 카드 펼침

- [ ] 카드 클릭 → 카드 하단이 펼쳐지며 상세 정보 노출
  - 현재 포인트
  - 누적 결제 (만원 단위)
  - 방문 횟수
- [ ] 우측 상태 라벨이 "상세 ▼" → "접기 ▲"
- [ ] 하단 액션 버튼 2개: "☕ 적립하기" (다크), "🎁 포인트 사용" (테두리)
- [ ] 다른 카드 클릭 시 이전 카드는 자동으로 접힘 (한 번에 한 개만 펼침)

## 5. 시나리오: 사장님 대시보드 (탭 📊)

진입: 사장님 로그인 후 📊 대시보드 탭

### 5.1 기본 표시

- [ ] 제목 "오늘의 가게" + "오늘도 좋은 하루 보내세요 ☕"
- [ ] **KPI 카드 3개** (가로 균등 분할)
  - 적립 건수 `18 건` (다크)
  - 지급 포인트 `4,280 P` (카라멜 강조)
  - 오늘 매출 `8.6 만원`
- [ ] **이번 주 매출** 막대 그래프
  - 7개 막대 (월~일)
  - 일요일(오늘)은 카라멜 그라디언트 + 굵은 글씨
  - 다른 요일은 옅은 베이지 막대
  - 가장 높은 토요일이 최대 높이
- [ ] **단골 손님 TOP 3** 카드 (1위는 카라멜 원 + 번호, 2~3위는 베이지)
- [ ] **가게 정보** 카드 (이름, 적립률, 리워드 기준)
- [ ] 하단 **로그아웃** 버튼 (테두리만 있는 미니멀 버튼)

### 5.2 로그아웃

- [ ] 로그아웃 클릭 → 로그인 화면으로 복귀
- [ ] 탭바 사라짐
- [ ] 다시 로그인하면 적립 탭으로 시작 (마지막 탭 상태가 리셋됨)

## 6. 시나리오: 손님 랜딩 화면

진입: 토글 → 📱 손님

### 6.1 초기 진입

- [ ] 최상단에 큰 그라디언트 카드 (다크 커피 톤) + ☕ + "진의 카페" + "단골 포인트 적립"
- [ ] 그 아래 인사말 + 전화번호 입력 카드
- [ ] **전화번호 입력창에 자동 포커스** (커서가 깜빡임)
- [ ] CTA "내 포인트 보기 →" 비활성 (회색)
- [ ] "5%가 적립돼요. 5,000P가 모이면 음료 한 잔 무료 ☕" 안내 카드
- [ ] 하단 **DEMO** 라벨 + 예시 칩 2개 (단골 / 신규)

### 6.2 데모 빠른 입력

- [ ] **단골 손님 칩** (`010-2345-7788 · 3,420P 보유`) 클릭 → 입력란에 자동 채움

- [ ] 칩에 hover 시 테두리가 카라멜색으로 변경
- [ ] 칩 active 시 살짝 작아지는 피드백

### 6.3 직접 입력

- [ ] `01012345678` 입력 → `010-1234-5678` 자동 포맷
- [ ] 유효 길이 도달 시 우측 "확인됨" 표시 + CTA 활성

### 6.4 조회 이동

- [ ] 유효한 번호 입력 후 CTA 클릭 → 포인트 조회 화면으로 전환
- [ ] Enter 키로도 submit 가능 (폼 동작)

## 7. 시나리오: 손님 포인트 조회 화면

진입: 랜딩에서 전화번호 입력 → "내 포인트 보기" 클릭

### 7.1 단골 진입 (`010-2345-7788`)

- [ ] 헤더 좌측: ☕ + "진의 카페 · 내 포인트"
- [ ] 헤더 우측: ↻ "다른 번호로 조회" 버튼
- [ ] 인사: `010-****-7788 손님 👋`
- [ ] 부제: "2026년 3월부터 함께해주셔서 감사합니다"
- [ ] **메인 카드** (다크 커피 그라디언트, 큰 그림자)
  - [ ] 큰 숫자 `3,420 P` (크림색, 카라멜 P)
  - [ ] "다음 무료 음료까지 1,580P 남음"
  - [ ] 프로그레스 바 약 68% 채움 (카라멜 그라디언트)
  - [ ] 도장 카드 5칸:
    - 1~3번째 칸: 카라멜 채움 (☕)
    - 4번째 칸: 점선 + 옅은 카라멜 부분 채움 (≈42%)
    - 5번째 칸: 점선만 (비어있음)
  - [ ] "도장 카드 · 3/5개" 라벨
- [ ] 안내 카드: "5%가 적립돼요. 5,000P 모이면 음료 한 잔 무료"
- [ ] 최근 내역 리스트 5건:
  - [ ] 적립 4건은 ☕ 아이콘 + 카라멜 `+xxxP`
  - [ ] 사용 1건은 🎁 아이콘 + 다크 `-5,000P` + "아메리카노 무료 사용"

### 7.2 신규 손님 진입 (`010-5555-1111` 또는 등록 안 된 임의 번호)

- [ ] 부제: "오늘 처음 만나뵙네요. 환영합니다 🌱"
- [ ] 메인 카드: `0 P`
- [ ] 다음 무료 음료까지 `5,000P 남음`
- [ ] 도장 카드 5칸 **모두 비어있음**
- [ ] 최근 내역: 점선 박스 empty state
  - [ ] 🌱 + "아직 적립 내역이 없어요"
  - [ ] "카운터에서 첫 적립을 시작해보세요 ☕"
  - [ ] "전체 보기" 버튼 숨김

### 7.3 다른 번호 조회

- [ ] 헤더 우측 ↻ 버튼 클릭 → 랜딩 화면으로 돌아감
- [ ] 입력 필드가 비워진 상태로 시작

## 8. 디자인 톤 점검

각 화면에서 톤이 일관되는지 확인합니다. ([ADR-0005](./adr/0005-design-tone.md) 기준)

- [ ] 배경: 크림 베이지 (`#fbf7f1`)
- [ ] 메인 텍스트: 다크 커피 (`#3d2817`)
- [ ] 액센트(카라멜 `#d97338`)는 **메인 CTA / 포인트 숫자 / 적립 표시**에만 사용
- [ ] 그린(`#7a9e5b`)은 **"확인됨" / "신규 손님" 뱃지**에만 사용
- [ ] 둥근 모서리: 카드 `rounded-2xl`, 버튼 `rounded-2xl`, 칩 `rounded-full`
- [ ] 그림자: 부드러운 종이 그림자 (강하지 않게)
- [ ] 폰트: Pretendard (변수 폰트가 CDN에서 로드되는지 네트워크 탭에서 확인)
- [ ] 숫자에 `tabular-nums` 적용 (`010-2345-7788`, `3,420`, `5,400` 등 자리가 흔들리지 않음)

## 9. 반응형 / 디바이스

- [ ] 모바일 뷰포트(393px): 컨테이너가 가장자리에 딱 맞춤
- [ ] 데스크탑(1280px+): 컨테이너는 가운데 `max-w-md`로 정렬
- [ ] 전화번호/금액 입력 필드 탭 시 모바일에서 숫자 키패드 표시 (`inputMode="numeric"`)
- [ ] Safari iOS에서 입력 시 줌인되지 않음 (폰트 크기 ≥16px)
- [ ] 사장님 탭바가 화면 하단에 고정, safe-area-inset 반영 (홈 인디케이터에 안 가림)
- [ ] 탭 전환 시 스크롤 위치는 화면별로 독립적 (탭 간 전환은 컴포넌트 마운트/언마운트)

UI 목업 단계이므로 다음은 의도적으로 구현되지 않았습니다.

- 새로고침 시 모든 상태가 초기화됨 (메모리 상태만 사용)
- 사장님 적립 내역과 손님 포인트 잔액이 **연동되지 않음** (각각 독립 mock)
- 손님 데이터가 있는 번호는 `010-2345-7788` 하나뿐. 그 외 번호는 모두 신규 손님(`0P`)으로 처리됨
- 손님 조회 화면의 단골 리스트는 손님 조회 화면 전용 mock (적립 화면의 최근 적립 리스트와 별개 데이터)
- 다크 모드 미지원
- 실제 저장소(DB/localStorage) 미연결
- "전체 보기", ⚙ 설정 버튼, 적립/사용 액션 버튼 등은 시각 요소만 있고 동작하지 않음
- PIN은 데모용 `1234`로 하드코딩. 변경 불가

이 항목들은 동작하는 MVP 단계에서 별도로 다룹니다.

## 11. 회귀 점검 빠른 체크리스트

PR을 만들기 전, 화면 하나를 건드렸더라도 **모든 화면**을 한 번씩 돌려보고 아래 항목을 빠르게 확인합니다.

- [ ] 사장님 로그인: 데모 칩 자동 입력 → 로그인 → 적립 탭 도착
- [ ] 사장님 적립 화면에서 단골/신규 각각 1건씩 적립이 정상 동작
- [ ] 사장님 손님 조회에서 검색 / 카드 펼침이 모두 동작
- [ ] 사장님 대시보드의 KPI, 막대 그래프, 단골 TOP 3 표시 정상
- [ ] 대시보드 → 로그아웃 → 로그인 화면 복귀
- [ ] 손님 랜딩 → 단골 칩 → 조회 화면 진입까지 끊김 없이 연결
- [ ] 손님 조회 화면의 도장 카드가 잔액에 맞게 정확히 표시됨
- [ ] 헤더 ↻ 버튼이 랜딩으로 돌아감
- [ ] 데모 토글로 모드 전환 시 깨지지 않음 (콘솔에 에러 없음)
- [ ] 사장님 모드 → 손님 모드 → 사장님 모드 시 로그인 상태 유지
