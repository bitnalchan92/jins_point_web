# ADR-0008: 디자인 톤 재정의 — 달콤한 진스쿡 (크림 + 옐로우)

- **Status**: Accepted
- **Date**: 2026-06-28
- **Supersedes**: [0005](./0005-design-tone.md)

## Context

[ADR-0005](./0005-design-tone.md)는 "동네 카페"를 가정하고 커피 브라운 + 카라멜 액센트의 다크한 톤을 정의했다. 이후 제품 가정이 **김밥·샌드위치 전문점 "달콤한 진스쿡"**으로 구체화되면서, 카페 다크 톤은 매장 성격과 어긋났다.

클로드로 생성한 디자인 시안([`document/prototype.html`](../prototype.html))을 레퍼런스로 삼아 톤을 다시 잡았다. 시안의 방향:

- 밝은 크림 배경에 **옐로우(브랜드)**를 메인 강조로 — 분식/김밥집의 친근하고 밝은 분위기
- 텍스트는 다크 브라운이 아닌 **잉크(거의 검정에 가까운 짙은 갈색)**로 가독성 강화
- 메타포를 ☕(커피)에서 **🍙(김밥)**으로 교체

## Decision

디자인 토큰을 [`src/index.css`](../../src/index.css)의 `@theme` 블록에서 전면 교체한다.

**컬러 팔레트**

| 역할 | 토큰 | 값 |
|------|------|-----|
| 베이스 | `--color-cream` | `#fcf7ec` |
| 카드 표면 | `--color-card` | `#ffffff` |
| 경계선 | `--color-line` | `#eee3cf` |
| 메인 텍스트 | `--color-ink` | `#241b12` |
| 보조 텍스트 | `--color-ink-soft` | `#9a8975` |
| 브랜드 | `--color-brand` / `--color-brand-dark` | `#ffc81f` / `#f0a91a` |
| 옅은 면 | `--color-pale` / `--color-pale-soft` | `#fff3c9` / `#fffbec` |
| 성공/리워드 | `--color-leaf` / `--color-leaf-bg` | `#5e9c53` / `#e9f2e0` |
| 위험 | `--color-danger` | `#d8553a` |

**타이포그래피**

- 본문/디스플레이 모두 **Pretendard** (별도 디스플레이 폰트 미사용). 헤딩은 `font-extrabold`(800)로 강조.
- 포인트·금액 숫자에는 `tabular-nums`.

**형태 · 메타포**

- 배경: 상단에서 퍼지는 radial-gradient(크림 → 옅은 베이지).
- 라운드: 카드 `rounded-[20~26px]`, 칩 `rounded-full`, 키패드 키 `rounded-[13px]`.
- 브랜드 로고: 🍙 옐로우 그라데이션 라운드 스퀘어 ([`src/ui/Logo.tsx`](../../src/ui/Logo.tsx)).
- 모션: `jp-fade` / `jp-pop` / `jp-shake` / `jp-toast` 키프레임을 `@theme`에 정의.

**원칙 (ADR-0005에서 계승)**

- 옐로우 강조는 핵심 정보(포인트, CTA)에 집중. 남용 금지.
- `leaf`(그린)는 성공/신규 상태에만.
- 다크 모드는 정의하지 않음 (밝은 카운터 가정).

## Consequences

좋은 점:

- 분식/김밥집의 밝고 친근한 분위기와 톤이 맞는다
- 옐로우 단일 강조로 "포인트" 정보가 한눈에 들어온다
- 토큰이 `@theme` 한 곳에 모여 있어 다음 톤 조정도 한 번에 가능

나쁜 점 / 따라오는 후속 결정:

- ADR-0005 기준으로 작성된 과거 문서/스크린샷은 톤이 어긋난다 → README·진행 문서를 함께 갱신
- 이모지(🍙, 🌱, 🎁)를 시각 언어로 차용. 디자인 시스템이 커지면 SVG 아이콘으로 교체 시점이 온다
- 헤드리스/이모지 폰트 없는 환경에서 🍙가 박스로 보일 수 있음 (실 브라우저에서는 정상)
