# ADR-0007: 사장님 인증 — 4자리 PIN + 디바이스 신뢰 모델

- **Status**: Superseded by [ADR-0014](./0014-production-auth-and-authorization.md)
- **Date**: 2026-06-27

## Context

[ADR-0002](./0002-phone-based-identification.md)에서 손님에게는 인증을 요구하지 않기로 결정했다. 사장님은 다르다 — 사장님 화면은 적립(매장 자산 증가), 포인트 사용(매장 자산 감소), 손님 데이터 조회 같은 민감한 작업을 한다.

요구사항:

- 카운터 환경에서 **수십 번 반복해도 부담스럽지 않은** 인증
- 직원 교대 시에도 빠르게 인계 가능
- 매장의 단일 디바이스(태블릿/POS 옆 폰)에서 주로 사용된다는 가정

대안:

- **이메일/비밀번호**: 보안성은 높지만, 카운터에서 매번 긴 문자열 입력은 비현실적. 직원 교대 시 비밀번호 공유 문제도 생긴다.
- **소셜 로그인 (카카오/구글)**: 첫 설정은 편하지만 매번 OAuth 플로우는 카운터에 맞지 않다.
- **PIN**: 4자리 숫자. 입력이 빠르고, 카운터 디바이스가 한 명 사장님 손에 있다는 가정 하에서 합리적.
- **생체 인증(지문/Face ID)**: 빠르지만 디바이스 의존이 크고, 직원 교대가 어려워진다.

## Decision

- **4자리 숫자 PIN**으로 사장님 모드를 보호한다.
- 인증 단위는 **세션(=현재 디바이스 사용 중인 시간)** 이다. 한 번 로그인하면 명시적으로 로그아웃하기 전까지 유지된다.
- 데모 단계의 PIN은 `1234`로 고정한다. 실제 MVP에서는 가게 설정에서 변경 가능하게 한다.
- 사장님 메인은 로그인 후 진입하며, 하단 탭바로 [적립 / 손님 조회 / 대시보드]를 전환한다.
- 로그아웃은 **대시보드 화면**에 둔다 (자주 누르는 곳이 아니라 의도적인 동선).

> 이 결정은 Stage 1 프로토타입에만 적용한다. 프로덕션 인증은
> [ADR-0014](./0014-production-auth-and-authorization.md)의 이메일·강한 비밀번호 +
> 필수 TOTP MFA 방식으로 대체했다.

## Consequences

좋은 점:

- 카운터에서 빠른 입력 가능 (숫자 키패드만)
- 직원 교대 시 PIN만 전달하면 됨
- 단순한 모델이라 첫 MVP가 가벼움

나쁜 점 / 따라오는 후속 결정:

- PIN은 보안 강도가 낮다 (10,000개 조합) — 매장 디바이스의 물리적 통제에 의존한다
- 다중 사장님/직원 권한 분리가 안 된다. 모두 같은 PIN을 공유. 권한 분리가 필요하면 별도 ADR ([[adr-owner-roles]] 후보)
- 본사/체인 환경에서는 디바이스 신뢰 모델이 깨진다 — [ADR-0003](./0003-single-store-scope.md)의 매장 다중화 결정과 함께 다시 봐야 한다
- 디바이스 분실 시 즉시 잠금 수단이 없다 — MVP에서 원격 로그아웃 기능 추가 검토 ([[adr-remote-revoke]] 후보)
- PIN을 잊었을 때 복구 플로우가 별도로 필요 (SMS/이메일 등) ([[adr-pin-recovery]] 후보)
