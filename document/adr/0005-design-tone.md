# ADR-0005: 디자인 톤 — 따뜻한 카페 감성

- **Status**: Accepted
- **Date**: 2026-06-27

## Context

같은 기능이라도 디자인 톤에 따라 제품의 인식이 완전히 달라진다. 이번 프로젝트는 동네 카페/디저트 매장이 사용하는 것을 가정하고 있어, 톤이 매장의 분위기와 어울려야 한다.

대안:

- **차갑고 모던** — 핀테크/쿠팡 같은 느낌. 효율적이지만 카페 매장과 톤이 어긋남
- **고급스러움** — 다크 베이스 + 골드. 객단가가 높은 디저트 카페에는 어울리지만 친근감이 떨어짐
- **따뜻하고 친근함** — 크림 베이스 + 커피 브라운 + 카라멜 액센트. 동네 단골 손님과의 관계를 시각화

## Decision

디자인 토큰을 `src/index.css`의 `@theme` 블록에 정의한다.

**컬러 팔레트**

- **베이스 (cream)** — `#fdfaf5`, `#fbf7f1`, `#f5ecdd`, `#e8ddd0`: 종이 같은 부드러운 배경
- **텍스트 (coffee)** — `#3d2817` (메인), `#5d4226`, `#8b6f47` (서브): 라떼 그라데이션처럼
- **액센트 (caramel)** — `#d97338` (메인), `#e89968`, `#b85a26`: 적립 포인트, 메인 CTA에만 사용
- **부가 (leaf)** — `#7a9e5b`: "확인됨", "신규 손님" 같은 성공 상태에만

**타이포그래피**

- **본문**: Pretendard variable
- **숫자/디스플레이**: Gmarket Sans → Pretendard 폴백
- **숫자에는 `tabular-nums`** 적용 (포인트 숫자가 흔들리지 않도록)

**형태**

- **둥근 모서리**: 카드 `rounded-2xl`, 버튼 `rounded-2xl`, 작은 칩 `rounded-full`
- **그림자**: 부드러운 종이 그림자 (`--shadow-soft`, `--shadow-lift`)
- **메타포**: "도장 쿠폰" 카페 감성을 시각 언어로 차용 — ☕ 이모지, 카라멜 색 강조

**원칙**

- 카라멜은 **메인 CTA와 포인트 숫자에만** 사용한다. 남용하면 강조 효과가 사라진다.
- 그린(leaf)은 **성공 상태**에만 사용한다. 일반 텍스트에 사용하지 않는다.

## Consequences

좋은 점:

- 매장의 분위기와 어울려, 사장님이 들고 다니기에 어색하지 않다
- 액센트 컬러 사용을 좁게 제한해서 "적립 포인트" 같은 핵심 정보가 한눈에 들어온다
- 디자인 토큰이 한 곳에 모여 있어 톤 조정이 한 번에 가능

나쁜 점 / 따라오는 후속 결정:

- 다크 모드는 정의하지 않음. 카운터에서 환한 조명을 가정 — 야간/술집 톤이 필요하면 별도 ADR
- 외부 폰트 CDN 의존 (Pretendard). 오프라인/사내망 환경에서는 폴백이 작동해야 한다
- 이모지(☕, ✨, 🌱)를 시각 언어로 차용. 디자인 시스템이 커지면 SVG 아이콘으로 교체할 시점이 온다
