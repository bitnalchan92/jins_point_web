# TypeScript 전환 설계

## 목표

현재 Vite + React 애플리케이션의 JavaScript/JSX 소스를 strict TypeScript로 전환한다. 런타임 기능, 상태 흐름, 계산 로직, UI 구조, 스타일, 문구와 초기 데이터는 변경하지 않는다.

## 범위

- `src` 아래의 `.js` 파일을 `.ts`로, `.jsx` 파일을 `.tsx`로 전환한다.
- `vite.config.js`를 `vite.config.ts`로 전환한다.
- `index.html`의 진입점 경로를 `src/main.tsx`로 갱신한다.
- TypeScript와 필요한 설정 파일을 추가한다.
- `package.json`에 독립적인 타입 검사 명령을 추가한다.
- README와 진행 상황 문서에서 JavaScript 사용 및 TypeScript 미도입 상태를 현재 상태에 맞게 갱신한다.

다음 항목은 범위 밖이다.

- 컴포넌트 분리 또는 병합
- 함수명, 변수명, 데이터 구조의 리팩터링
- 비즈니스 규칙과 유효성 검사 변경
- 렌더링 마크업, CSS 클래스, 표시 문구 변경
- 신규 기능 또는 테스트 프레임워크 도입
- 의존성 버전 업그레이드

## 전환 방식

`strict: true`를 사용하는 전체 전환을 한 번에 수행한다. `allowJs`는 사용하지 않으며, 전환 완료 후 실행 소스에 `.js` 또는 `.jsx` 파일을 남기지 않는다.

TypeScript 컴파일은 Vite의 번들 변환과 분리해 `tsc --noEmit`으로 검사한다. Vite에 적합한 모듈 해석과 JSX 설정을 사용하고, 브라우저 애플리케이션 및 Vite 설정 파일에 필요한 타입 환경을 각각 구성한다.

## 타입 설계

### 공유 데이터

`src/lib/data.ts`에 다음 도메인 타입을 정의한다.

- 적립 및 사용 내역 종류
- 손님 내역 항목
- 손님
- 데모 입력 칩
- 대시보드 KPI 및 주간 데이터

기존 JSDoc 타입은 TypeScript 선언으로 치환한다. 상수 값과 데이터 내용은 유지한다.

### 공유 상태

`src/store.tsx`에 Context 값, 적립 로그, 포인트 적립 결과, 포인트 사용 결과, 손님 추가 결과 타입을 정의한다. Context 초기값은 기존처럼 `null`을 사용하고, `useStore`의 Provider 외부 호출 오류도 유지한다.

상태 초기값에서 빈 객체나 빈 배열로 인해 타입이 과도하게 좁아지는 부분만 명시적으로 타입을 부여한다. 포인트 계산, 손님 검색, 로그 순서, 최대 로그 개수와 반환 조건은 변경하지 않는다.

### 컴포넌트

각 컴포넌트의 props를 파일 가까이에 타입 또는 인터페이스로 선언한다. 상태 값은 가능한 경우 기존 초기값과 사용처에서 추론하게 두고, `null`, 문자열 리터럴 조합, 동적 객체처럼 추론이 불충분한 경우에만 제네릭 타입을 명시한다.

이벤트 핸들러에는 React 이벤트 타입을 사용한다. 화면 탭, 적립/사용 모드와 단계처럼 제한된 문자열 값은 유니온 타입으로 표현하되 기존 분기와 값은 유지한다.

`any`는 사용하지 않는다. 불가피한 외부 경계가 발견되면 `unknown`에서 안전하게 좁히며, 런타임 동작을 바꾸는 새 검증은 추가하지 않는다.

## 파일 및 import 규칙

- 소스 파일 확장자는 역할에 따라 `.ts` 또는 `.tsx`를 사용한다.
- 내부 import는 현재 Vite 해석 방식과 동일하게 확장자를 생략한다.
- `src/main.tsx`의 `App.jsx` 명시 확장자는 제거한다.
- DOM 루트는 현재 `index.html`이 항상 `#root`를 제공한다는 전제 아래 타입을 확정한다.
- CSS와 정적 리소스는 변경하지 않는다.

## 동작 보존 원칙

변환 전후 다음 요소가 동일해야 한다.

- `/`와 `/admin` 라우팅 및 리다이렉트
- 사장님 로그인, 탭 이동과 로그아웃
- 손님 검색, 등록, 포인트 적립 및 사용
- 적립률 수정과 모든 계산 결과
- Context의 세션 메모리 상태
- 초기 손님, 대시보드 및 내역 데이터
- 렌더링 마크업, 클래스명, 문구와 화면 레이아웃

타입 오류 해결을 위해 기존 코드의 의미를 변경하지 않는다. 타입 모델이 실제 코드와 충돌하면 실제 런타임 동작을 표현하도록 타입을 조정한다.

## 검증

변환 전 기준선과 변환 후 결과를 다음 명령으로 비교한다.

```bash
npm run build
npm run lint
```

전환 후 다음 검사를 추가한다.

```bash
npm run typecheck
```

또한 개발 서버에서 최소 회귀 확인을 수행한다.

- `/` 손님 화면 렌더링 및 등록 손님 조회
- `/admin` 로그인
- 포인트 탭, 손님 관리 탭, 대시보드 탭 렌더링
- 기존 수동 테스트 가이드의 핵심 적립 및 사용 흐름

자동화 테스트가 현재 없고 이번 변경은 런타임 기능 추가가 아닌 정적 타입 전환이므로 테스트 프레임워크는 새로 도입하지 않는다. 검증 실패 시 기능 코드를 재설계하지 않고 타입 선언이나 설정을 실제 동작에 맞게 수정한다.

## 완료 조건

- 실행 소스와 Vite 설정에 `.js` 또는 `.jsx` 파일이 없다.
- `strict: true` 상태에서 `npm run typecheck`가 통과한다.
- `npm run lint`와 `npm run build`가 통과한다.
- 핵심 수동 흐름에서 변환 전과 다른 동작이나 화면이 발견되지 않는다.
- `any` 또는 동작 우회를 위한 광범위한 타입 단언이 없다.
